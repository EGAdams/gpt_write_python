Agent System Overview

Our system uses "Agents" as a key component. Here's a general breakdown:

Agents

These are the core of the application logic. They receive user input and a list of previous steps taken, and return either an "AgentAction" or an "AgentFinish". The former corresponds to the tool to use and the input to that tool, while the latter signifies that the agent has completed its task and has information to return to the user.

Tools

These are the actions an agent can take. The tools available to an agent depend on what the agent is designed to do.

Toolkits

These are groups of tools designed for a specific use case. For instance, an agent that needs to interact with a SQL database might need one tool to execute queries and another to inspect tables.

Agent Executor

This wraps an agent and a list of tools. It is responsible for running the agent iteratively until a stopping criterion is met.

Types of Agents

The documentation also distinguishes between two types of agents:

  • Action Agents: These agents decide on an action to take and execute that action one step at a time.
  • Plan-and-Execute Agents: These agents first decide on a plan of actions to take, and then execute those actions one at a time. They are more suitable for complex or long-running tasks.
